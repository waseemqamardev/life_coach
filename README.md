# AI Life Navigator

AI-powered decision making app built with Flutter. Helps users define a decision, lay out options, answer guided questions, and receive a structured recommendation with risk score, pros/cons, outcome prediction, and a 3-step action plan.

## Stack

- Flutter 3.19+, Dart 3.3+
- **State**: `flutter_riverpod`
- **Navigation**: `go_router`
- **Storage**: `hive` for decisions/history, `shared_preferences` for lightweight prefs
- **UI**: `google_fonts` (Inter), `fl_chart`, custom widgets
- **Models**: Hive-backed (generated adapters under `*.g.dart`)

## Running

```bash
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs   # generates Hive adapters
flutter run

# Android support
The Android platform folder is now generated by `flutter create --platforms=android .` so the app can be built for both iOS and Android.
```

The first build_runner pass creates `decision.g.dart` and other generated adapters used by Hive.

## Screens

All 59 screens from the brief are implemented with proper navigation. The highest-traffic screens (splash, onboarding, auth, home, decision flow, results, history, settings, subscription) are built to closely match the designs. The remaining screens use the same design system tokens and layouts so they slot in cleanly with the same look.

## Design tokens

Centralised in `lib/core/theme/app_colors.dart` and `app_text_styles.dart`. To tweak the brand colors globally edit those two files вЂ” every widget reads from them via `Theme.of(context)` or directly from the `AppColors` class.

- Primary gradient: deep indigo `#3B4FE0` в†’ vibrant violet `#9747FF`
- Light bg: `#FFFFFF`, light surface: `#F7F8FA`
- Dark bg: `#0A0E1F`, dark surface: `#131A2E`
- Success: `#22C55E`, warning: `#F59E0B`, danger: `#EF4444`, info: `#3B82F6`

## Assets

The designs include several PNG illustrations (brain hologram, envelope-with-lock, robot, trophy, etc.). Drop them into `assets/images/` using the exact filenames listed in `lib/core/constants/app_assets.dart`. The app gracefully falls back to icon placeholders if a file is missing so it still runs without them.

## Notes for production

- **AI service is stubbed** in `lib/core/services/ai_service.dart` вЂ” swap the `Future.delayed` for a real API call (OpenAI / Anthropic / your backend).
- **Auth is local-only** вЂ” the sign-up / login screens just persist to SharedPreferences. Wire in Firebase Auth / Supabase / your own backend before shipping.
- **Payments** вЂ” the payment flow is fully UI; integrate `in_app_purchase` (Apple/Google) or Stripe before going live.
- **Permissions** вЂ” request notifications/location only when the user reaches the relevant feature. The Permission screen is a UI shell; use `permission_handler` to actually request.
